# 秒杀系统部署文档

## 1. 项目概述
- **项目名称**：秒杀系统
- **项目描述**：基于Go语言和Vue.js的高并发秒杀系统，采用微服务架构设计。后端使用Gin框架构建RESTful API，集成MySQL数据库、Redis缓存和RabbitMQ消息队列，实现商品库存管理、用户认证、订单处理等核心功能。前端采用Vue 3 + Vite构建现代化单页应用，提供用户友好的交互界面。


## 2. 环境准备
### 2.1 基础环境要求
- **云服务提供商**：本地开发环境 / 阿里云ECS
- **操作系统**：Windows 10 / CentOS 7
- **必要工具**：Docker、Docker Compose


### 2.2 技术栈版本说明
| 组件 | 版本 | 作用 |
|------|------|------|
| Go | 1.25.1 | 后端开发语言 |
| Gin | - | 后端Web框架 |
| Vue | 3.5.18 | 前端框架 |
| Vite | 7.0.6 | 前端构建工具 |
| MySQL | 8.0 | 数据持久化存储 |
| Redis | 7 | 缓存与分布式锁 |
| RabbitMQ | 3-management | 消息队列 |


## 3. 部署步骤
### 3.1 Docker环境搭建
1. 安装Docker：根据操作系统选择对应安装包，参考[Docker官方文档](https://docs.docker.com/get-docker/)完成安装
2. 安装Docker Compose：参考[Docker Compose安装指南](https://docs.docker.com/compose/install/)
3. 验证安装：
```bash
docker --version
docker-compose --version
```


### 3.2 系统部署（基于Docker Compose）
1. 创建部署目录并进入
```bash
mkdir seckill-deploy && cd seckill-deploy
```

2. 创建`docker-compose.yml`文件，内容如下：
```yaml
version: "3.9"
services:
  redis:
    image: redis:7
    container_name: redis_seckill
    ports:
      - "6379:6379"
  
  rabbitmq:
    image: rabbitmq:3-management
    container_name: rabbitmq_seckill
    ports:
      - "5672:5672"
      - "15672:15672"
    environment:
      - RABBITMQ_DEFAULT_USER=ez
      - RABBITMQ_DEFAULT_PASS=123456
  
  mysql:
    image: mysql:8
    container_name: mysql_seckill
    environment:
      - MYSQL_ROOT_PASSWORD=123456
      - MYSQL_DATABASE=seckill
    ports:
      - "3307:3306"
    volumes:
      - mysql_data:/var/lib/mysql
  
  app:
    build: .
    container_name: seckill_app
    ports:
      - "8080:8080"
    depends_on:
      - mysql
      - redis
      - rabbitmq

volumes:
  mysql_data:
```

3. 放置后端应用Dockerfile到当前目录（用于构建app服务）

4. 启动所有服务
```bash
docker-compose up -d
```

5. 验证服务状态
```bash
docker-compose ps
```


### 3.3 前端应用部署
1. 前端打包：在Vue项目根目录执行
```bash
npm run build
```

2. 获取打包产物：生成的静态文件位于`dist`目录

3. 部署方式：
   - 本地开发：通过`npm run dev`启动开发服务器（默认端口5175）
   - 生产环境：将`dist`目录部署到Nginx服务器，配置反向代理指向后端API（端口8080）


## 4. 配置详情
### 4.1 核心配置项
- **数据库配置**：MySQL连接字符串（包含用户名、密码、地址、端口）
- **Redis配置**：连接地址（默认localhost:6379）、密码（如配置）
- **RabbitMQ配置**：
  - 地址：localhost:5672
  - 用户名：ez
  - 密码：123456


### 4.2 端口映射说明
| 服务 | 容器内部端口 | 宿主映射端口 |
|------|--------------|--------------|
| 前端应用 | - | 5175 |
| 后端服务 | 8080 | 8080 |
| MySQL | 3306 | 3307 |
| Redis | 6379 | 6379 |
| RabbitMQ | 5672（服务）、15672（管理界面） | 5672、15672 |


## 5. 系统验证
### 5.1 服务可用性检查
1. 后端API验证：访问`http://localhost:8080/health`检查服务状态
2. 数据库连接：使用MySQL客户端连接`localhost:3307`，验证数据库是否正常
3. Redis连接：使用`redis-cli`连接`localhost:6379`，验证缓存服务
4. RabbitMQ管理界面：访问`http://localhost:15672`，使用账号密码（ez/123456）登录验证


### 5.2 功能测试
1. 用户注册/登录：通过前端界面或API测试用户认证功能
2. 商品浏览：验证商品列表接口是否正常返回数据
3. 秒杀下单：模拟高并发场景，测试秒杀功能和库存控制
4. 订单查询：验证订单状态查询功能


## 6. 日常维护
### 6.1 日志查看
```bash
# 查看后端应用日志
docker logs -f seckill_app

# 查看数据库日志
docker logs -f mysql_seckill
```

### 6.2 服务重启
```bash
# 重启单个服务
docker-compose restart app

# 重启所有服务
docker-compose restart
```

### 6.3 数据备份
```bash
# MySQL数据备份
docker exec mysql_seckill mysqldump -u root -p123456 seckill > seckill_backup.sql
```
