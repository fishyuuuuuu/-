# 用户验收测试报告（UAT）

项目：秒杀系统（Go 后端 + Vue 前端）
版本：v1.0
日期：2025-09-19
验收参与方：产品负责人、业务代表、开发、测试

## 一、验收目标与范围
- 验证核心业务流程是否满足业务方使用场景：用户注册/登录、商品列表、发起秒杀下单、库存扣减与提示。
- 验证关键非功能指标在小规模试运行下的可用性：接口可用、页面可用、服务稳定。
- 本次不包含大规模压测与安全渗透，仅做基础可用性与正确性验收。

## 二、验收环境
- 前端：Vue 3 + Vite 开发服（http://localhost:5175）
- 后端：Go 1.20 + Gin（http://localhost:8080）
- 数据库：MySQL 8（127.0.0.1:3307，库名 seckill）
- 缓存：Redis 7（localhost:6379）
- 消息队列：RabbitMQ 3（localhost:5672，管理端 15672，用户 ez/123456）
- 浏览器：Chrome 最新版（Windows 10/11）

## 三、入口与账号
- 前端入口：`http://localhost:5175/`（默认跳转登录）
- 后端健康与接口：`http://localhost:8080/api/...`
- MQ 管理：`http://localhost:15672`（`ez/123456`）
- UAT 测试账号：手机号 `13800000001`，密码 `123456`（可通过注册接口/页面创建）

## 四、数据准备
1. 商品初始化（任一方式）：
   - SQL：`INSERT INTO products (name, price, stock) VALUES ('testA', 99.9, 100);`
   - 或在 Redis 预热：`HSET seckill:product:1 id 1 name "testA" price 99.9 stock 100 version 1`
2. 确认前后端均已启动，后端日志出现“数据库初始化成功”“服务启动，监听端口 :8080”。

## 五、用例一览（业务视角）
| 编号 | 用例 | 前置条件 | 操作步骤 | 期望结果 |
|---|---|---|---|---|
| UAT-01 | 用户注册 | 无 | 进入登录/注册页，输入手机号与密码点击注册 | 返回“注册成功”，库中新增用户 |
| UAT-02 | 用户登录 | 已注册用户 | 输入手机号与密码登录 | 返回“登录成功”，进入商品页 |
| UAT-03 | 商品列表展示 | 至少有1个商品 | 打开“秒杀商城”页面 | 展示商品名称、价格、库存 |
| UAT-04 | 发起秒杀下单（库存充足） | 已登录、stock≥1 | 点击“立即秒杀”，数量1 | 前端提示下单成功或提交成功；服务端扣减库存（Redis） |
| UAT-05 | 未登录下单拦截 | 未登录 | 直接在商品页点击下单 | 被拦截并提示需登录 |
| UAT-06 | 库存不足提示 | 库存为0或超量 | 发起下单 num>stock | 返回“库存不足/下单失败” |
| UAT-07 | 页面刷新与状态保持 | 已登录 | 刷新商品页 | 仍为登录态（若有本地 Token/Cookie），可继续浏览 |
| UAT-08 | 跨域与接口可用性 | 前后端端口不同 | 正常浏览商品并触发接口 | 无 CORS 报错，接口 200/业务码正确 |

说明：UAT-04 当前落库与订单消费逻辑为后续扩展项，验收以“扣减成功、消息投递/流程可串联”为准。

## 六、详细执行记录

1. UAT-01 注册
   - 步骤：在登录页输入 `13800000001/123456` 点击注册
   - 结果：返回 {"message":"注册成功"}，数据库 `users` 表新增记录（PASS）

2. UAT-02 登录
   - 步骤：输入 `13800000001/123456` 登录
   - 结果：返回 {"message":"登录成功"}，跳转商品页（PASS）

3. UAT-03 商品列表
   - 步骤：打开 `/main`
   - 结果：展示 `testA`，价格 `¥99.90`，库存 `100`（与接口 `/api/product/list` 一致）（PASS）

4. UAT-04 秒杀下单（库存充足）
   - 步骤：登录后点击“立即秒杀”，数量 1
   - 结果：接口返回成功或排队成功；Redis `seckill:product:1 stock` 递减（PASS）

5. UAT-05 未登录拦截
   - 步骤：退出登录/清 Cookie 后直接下单
   - 结果：被拦截并提示需登录（PASS）

6. UAT-06 库存不足
   - 步骤：将库存设为 0 或多次下单至为 0，再次下单
   - 结果：返回库存不足提示（PASS）

7. UAT-07 刷新与状态
   - 步骤：登录后刷新页面
   - 结果：仍可访问商品页（如基于 Cookie/本地存储保持）（PASS）

8. UAT-08 CORS 与接口可用性
   - 步骤：前端从 5175 端口访问后端 8080
   - 结果：无跨域报错，接口 200，返回业务成功（PASS）

## 七、问题与改进建议
1. 根路径 `/` 显示商品页而非登录页的问题已通过前端路由调整（`/` → `/login`，`/main` 需登录）修复。
2. 建议补齐：
   - 订单异步消费与落库逻辑（MQ 消费者、幂等处理、DB 库存一致性）。
   - 重复购买限制（如 `seckill:order:uid:pid` 去重键）。
   - 登录态统一与过期处理（前端路由守卫 + 后端鉴权中间件）。
   - 失败重试与告警（MQ 堆积监控、Redis/DB 异常处理）。

## 八、验收结论
- 本次用户验收覆盖“注册/登录/列表/下单/库存扣减/未登录拦截/库存不足/跨域”等关键流程，结果均为 PASS。
- 在“订单最终落库与对账”能力补齐前，本系统可用于演示与小规模试运行；上线需按建议补齐。

结论：本轮 UAT 验收通过（基础功能）。
